- **404 페이지 없음**: 잘못된 경로 접근 시 빈 화면이 표시됩니다.
- **Footer 없음**: 페이지 끝이 Contact 섹션으로 단절되어 마무리감이 부족합니다.

| SectionHeader 중복 코드 | ⚠️ 개선 필요 |
| 폰트 로딩 | ⚠️ 개선 필요 |

### 잘 된 점
- `index.css`의 CSS 변수 체계가 전 컴포넌트에 일관되게 적용되어 있습니다.
- MUI `theme.js`의 팔레트가 CSS 변수와 동일한 값으로 정렬되어 있습니다.
- 대부분의 버튼이 `borderRadius: '980px'` pill 형태로 통일되어 있습니다.

### 개선 필요
- **ContactInfo SNS 버튼 불일치**: `borderRadius: 1.5`(12px)로 다른 모든 버튼의 pill 스타일과 다릅니다.
- **SectionHeader 코드 중복**: `home-page.jsx`에 `SectionHeader` 컴포넌트가 있지만, `about-page.jsx`와 `projects-page.jsx`는 각자 동일한 헤더 마크업을 직접 구현하고 있습니다. `components/common/`으로 분리가 필요합니다.
- **Roboto 폰트 미로딩**: `theme.js`에 `fontFamily: '"Roboto"'`로 지정했지만 `index.html`에 Google Fonts import가 없어 시스템 폰트로 fallback됩니다.

## 4. 성능 & 반응형

| 항목 | 상태 |
|---|---|
| 접근성 (ARIA) | ✅ 완료 |
| prefers-reduced-motion | ✅ 완료 |
| React 최적화 (memo, useMemo) | ✅ 완료 |
| Hero 반응형 | ✅ 완료 (2026-03-05 업데이트) |
| 이미지 lazy loading | ⚠️ 개선 필요 |
| 라우트 코드 분할 | ⚠️ 개선 필요 |
| 모바일 높이 하드코딩 | ⚠️ 개선 필요 |
| Guestbook 이모지 접근성 | ⚠️ 개선 필요 |

### 잘 된 점
- `ProjectCard`의 스켈레톤 shimmer + 에러 fallback 처리가 완성도 높습니다.
- `React.memo(SkillCard)`, `useMemo`, `useCallback`으로 불필요한 리렌더링이 방지되어 있습니다.
- `aria-label`, `role`, `aria-controls` 등 접근성 마크업이 충실합니다.

### 개선 필요
- **이미지 lazy loading 없음**: `ProjectCard`의 `<Box component="img">`에 `loading="lazy"` 속성이 없어 모든 이미지를 즉시 로딩합니다.
- **라우트 코드 분할 없음**: `App.jsx`가 `HomePage`, `AboutPage`, `ProjectsPage`를 모두 즉시 import합니다. `React.lazy()` + `Suspense`로 초기 번들 크기를 줄일 수 있습니다.
- **`projects-page.jsx` 높이 하드코딩**: `minHeight: 'calc(100vh - 64px)'` — MUI AppBar는 모바일에서 56px이므로 약 8px 오차가 발생합니다.
- **Guestbook 이모지 버튼 접근성**: 이모지 선택 영역이 `<Box onClick>`으로 구현되어 키보드 접근이 불가능합니다. `<button>` 태그로 변경이 필요합니다.
